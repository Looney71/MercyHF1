# Диспетчер задач MercyHF AI (n8n Workflow)

Этот workflow предназначен для автоматического создания структурированных GitHub Issues для задач разработки MercyHF. Это позволяет ИИ-агентам (таким как Jules и Antigravity) работать с четко сформулированными задачами, соответствующими стандартам проекта.

---

## 🚀 Инструкция по импорту в n8n

Вы можете легко импортировать этот workflow в свой экземпляр n8n:

1. Откройте веб-интерфейс **n8n**.
2. В левом меню выберите **Workflows**.
3. Нажмите кнопку **+ Add workflow** (или создайте пустой workflow).
4. В правом верхнем углу нажмите на троеточие (`...`) и выберите **Import from file** (Импорт из файла).
5. Выберите файл `MercyHF_AI_Task_Dispatcher.workflow.json` из этой папки.
6. Вы также можете просто открыть JSON-файл, скопировать весь его контент и нажать `Ctrl + V` прямо на холсте редактора n8n.

---

## 🔑 Настройка учетных данных GitHub

Для работы узла **GitHub Create Issue** необходимо настроить авторизацию:

1. Дважды кликните по узлу **GitHub Create Issue** на холсте.
2. В поле **Credential for GitHub API** выберите или создайте новое подключение.
3. Чтобы создать новое подключение:
   * Нажмите **Create New Credential**.
   * Выберите метод авторизации: **Access Token** (рекомендуется) или **OAuth2**.
   * Если вы используете **Access Token**, сгенерируйте Personal Access Token (PAT) в настройках вашего GitHub-аккаунта (`Settings` -> `Developer settings` -> `Personal access tokens` -> `Tokens (classic)`) с правами `repo`.
   * Вставьте токен в поле **Access Token** в n8n и сохраните учетные данные.
4. В параметрах узла укажите правильные значения для:
   * **Repository Owner** (ваш логин на GitHub или название организации).
   * **Repository Name** (название репозитория, например `MercyHF`).

---

## 📝 Редактирование полей задачи

Вся информация о задаче настраивается в узле **Set Task Fields** (узел типа *Set*):

1. Откройте узел **Set Task Fields**.
2. Измените значения следующих полей по умолчанию для создания нужной задачи:
   * `task_title` — Название задачи (например, `INDEX-0: Build first playable visual blockout`).
   * `task_description` — Подробное описание цели задачи.
   * `task_type` — Тип задачи (`visual`, `logic`, `audio` и т.д.).
   * `target_agent` — Имя целевого агента (`antigravity` или `jules`).
   * `priority` — Приоритет задачи (`low`, `medium`, `high`, `critical`).
   * `level` — Название уровня (`index-0` и др.).
3. Эти параметры автоматически подставятся в шаблон GitHub Issue и сгенерируют соответствующие метки (Labels).

---

## ⚡ Ручной запуск workflow

После импорта и настройки вы можете запустить процесс вручную:

1. Откройте workflow в редакторе n8n.
2. Убедитесь, что все поля в **Set Task Fields** заполнены нужной информацией.
3. Нажмите кнопку **Listen for test event** или **Execute workflow** внизу экрана.
4. Вы также можете нажать на кнопку **Execute Node** непосредственно на узле **Manual Trigger**.
5. Узел GitHub создаст задачу, и вы увидите ответ от GitHub API в панели результатов.

---

### Метки (Labels), создаваемые автоматически:
Workflow автоматически вешает на Issue следующие теги:
* `agent:<target_agent>` (например, `agent:antigravity`)
* `type:<task_type>` (например, `type:visual`)
* `level:<level>` (например, `level:index-0`)
* `priority:<priority>` (например, `priority:high`)
