## 2024-05-30 - Replace UGameplayStatics::GetAllActorsOfClass with TActorRange for performance
**Learning:** In Unreal Engine, `UGameplayStatics::GetAllActorsOfClass` allocates a new TArray and iterates all actors in the world every time it is called. Since a game typically can have hundreds or thousands of actors, frequently calling `GetAllActorsOfClass` causes unnecessary allocations and iterations, especially when done in BeginPlay for caching. Unreal Engine provides `TActorRange<AActor>(GetWorld())` which uses an iterator instead of allocating a TArray, which can be faster and more memory efficient when simply caching specific actors during BeginPlay or initialization, but still requires iterating over all actors. An even more optimal approach for caching logic that is looking for specific actor types or tags would be to only gather the specific classes needed (e.g. `TActorRange<AMercySystemTextActor>`).

**Action:** Replace `UGameplayStatics::GetAllActorsOfClass(GetWorld(), AActor::StaticClass(), AllActors);` with `TActorRange<AActor>(GetWorld())` in caching functions or replace `GetAllActorsOfClass` entirely where appropriate with more specific types.
